# MonScanner Compliance Mapper

Don't sell security. Sell compliance. When legal says "the regulation requires independent testing," that's mandatory budget. Position MonScanner as the required control.

## PCI-DSS 4.0 (FinTech, E-commerce)
- 11.4.1 — Annual pen test → MonScanner Full Pen Test
- 11.3.1/11.3.2 — Quarterly vuln scans → Cloud Security Assessment
- 6.4.1 — Web app protection → Application Pen Test
- 12.10.2 — IR plan testing → Red Team Exercise
**Hook**: "PCI-DSS 4.0 enforcement deadline has passed. Your QSA will ask for 11.4.1 evidence."

## SOC 2 Type II (SaaS, Cloud)
- CC6.1 — Logical access → Cloud Assessment (IAM review)
- CC6.6 — System boundary → External Pen Test
- CC7.1 — Unauthorized change detection → Red Team Exercise
- CC8.1 — Change management → Application Pen Test (CI/CD)
**Hook**: "Your auditor requires pen test evidence under CC6.6. We deliver the format they accept."

## DORA (EU Financial Services)
- Art. 24 — Threat-Led Penetration Testing → Red Team (TIBER-EU aligned)
- Art. 25 — ICT system testing → Application + Cloud Assessment
- Art. 26 — Independent qualified testers → MonScanner credentials (OSCP/OSCE)
- Art. 28 — Third-party ICT risk → Vendor Security Assessment
**Hook**: "DORA requires TLPT for critical financial entities. Article 26 mandates independent qualified testers."

## SEC Cybersecurity Disclosure (US Public Companies)
- Material incident disclosure (4 days) → Red Team + IR Planning
- Annual risk management disclosure → Comprehensive Assessment for 10-K
- Board cybersecurity expertise → vCISO Retainer
**Hook**: "SEC rules require annual cybersecurity disclosure. Our report provides 10-K evidence."

## EU AI Act (AI Companies)
- Art. 9 — Risk management for high-risk AI → AI System Security Assessment
- Art. 15 — Accuracy, robustness, cybersecurity → App Pen Test + AI Testing
- Art. 42 — Conformity assessment → Comprehensive Assessment
**Hook**: "EU AI Act requires cybersecurity testing for high-risk AI under Article 15. We test prompt injection, data poisoning, model extraction."

## NIS2 (EU Critical Infrastructure)
- Art. 21 — Cybersecurity risk management → Comprehensive Assessment
- Art. 21(2)(e) — Network security → Cloud + Network Pen Test
- Art. 23 — Incident reporting (24hr/72hr) → Red Team + IR Testing
**Hook**: "NIS2 requires 24-hour incident reporting. Our red team tests whether your team can meet that window."

## HIPAA (Healthcare)
- §164.308(a)(8) — Security evaluation → Comprehensive Assessment
- §164.312(e)(1) — Transmission security → Network + API Pen Test
- §164.306(a)(2) — Protect against threats → Red Team Exercise
**Hook**: "HIPAA requires periodic security evaluations. Our assessment covers ePHI systems end-to-end."
